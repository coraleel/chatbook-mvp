# Deep Research Report: ChatBook MVP

## Цель
Исследование инструментов и подходов для создания MVP системы вопросно-ответных чатов по документам.

## Ключевые выводы
1.  Для быстрого прототипа оптимален подход RAG (Retrieval-Augmented Generation).
2.  Локальная работа без API-ключей возможна с использованием локальных эмбеддинг-моделей (например, из библиотеки `sentence-transformers`) и локальных LLM.
3.  Cursor с Agent Mode может автономно генерировать большую часть кода, что ускоряет разработку.

## Выбранный стек для MVP
*   **Язык:** Python
*   **Основные библиотеки:** `langchain` (оркестрация), `sentence-transformers` (эмбеддинги), `chromadb` (векторная БД), `streamlit` (веб-интерфейс, опционально)
*   **Инструмент разработки:** Cursor IDE

## Риски и "белые пятна"
*   **Боль #1:** Обработка PDF может требовать дополнительных библиотек (PyPDF2, pdfplumber) и очистки текста.
*   **Боль #2:** Размер контекста модели. Текст книги необходимо разбивать на чанки.
*   **Решение:** Начать с простого `.txt` файла для валидации конвейера.
