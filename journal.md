# Лабораторный журнал

## 1. Подготовка данных
*   **Магия:** Удалось быстро подготовить "грязные данные": установить Cursor, создать папку `chatbook-mvp` и, используя Блокнот, сохранить главу книги в идеальном для обработки формате `.txt`.
*   **Боль:** На этом этапе явных технических проблем не возникло. Основной вызов был концептуальным: *"Какой формат данных выбрать для максимально простого старта, чтобы не увязнуть в проблемах парсинга?"*
*   **Решение:** Принял стратегическое решение начать с `.txt`, созданного в Блокноте, вместо более распространённого, но сложного PDF. Это решение, основанное на принципе KISS, полностью оправдалось и позволило избежать "боли" на следующем этапе (чтение файла в коде прошло гладко).

## 2. Запуск промпта в Cursor
*   **Магия:** Cursor в Agent Mode сам создал проектную структуру, файл `requirements.txt` и начал писать код на Python. Он понял задачу!
*   **Боль:** Первый запуск скрипта упал с ошибкой `ModuleNotFoundError: No module named 'langchain'`.
*   **Решение:** Подсказал Cursor в чате: "Установи необходимые зависимости из requirements.txt". Он выполнил команду `pip install -r requirements.txt` в терминале.

## 3. Первый запуск и ошибка
*   **Магия:** Cursor успешно создала файл `chatbook.py` со всей запрошенной структурой.
*   **Боль:** При запуске `python chatbook.py` возникает ошибка: `[Вставьте сюда краткое описание ошибки, например, "ModuleNotFoundError: No module named 'nltk'"]`.
*   **Решение:** Скопировала текст ошибки в чат Cursor и попросила исправить код.

## 4. Взаимодействие с Agent Mode
*   **Магия:** Cursor автономно предложил создать папку `texts` для организации файлов.
*   **Боль:** Первый раз вижу запрос на разрешение выполнения команд. Непонятно, что делать.
*   **Решение:** Нажала `Allow` для создания папки, так как это соответствует логике проекта.

## 5. Первый тестовый запуск
*   **Магия:** Cursor автоматически перешёл к этапу тестирования, попытавшись запустить скрипт с ключом `--help`.
*   **Боль/Результат:** После выполнения команды в терминале появилось следующее:
    ```
    [ВСТАВЬТЕ СЮДА СКОПИРОВАННЫЙ ТЕКСТ ИЗ ТЕРМИНАЛА]
    ```
*   **Решение:** Ожидаю следующих действий Cursor (скорее всего, он проанализирует вывод и начнёт исправлять код).

## 6. Ревью сгенерированного кода
*   **Магия:** Cursor в Agent Mode самостоятельно предложил готовый пакет изменений: полный скрипт `chatbook.py` (~260 строк), документацию `README.md` и тестовый файл `example.txt`. Интерфейс ревью наглядно показывал все добавляемые и удаляемые строки.
*   **Боль:** Впервые увидела такой объём сгенерированных изменений. Возникли вопросы: «Как быстро проверить?», «Не пропустить ли ошибку?», «Что если нажать не ту кнопку?». Основная трудность — необходимость быстро принять решение по чужому (AI) коду.
*   **Решение:** Внимательно просмотрела ключевые изменения в `chatbook.py`: убедилась, что есть основные функции (`read_text_file`, `split_text`, `find_answer`), нет вызовов внешнего API (как требовалось), реализована обработка ошибок. 
Увидев, что структура соответствует техническому заданию из промпта, нажала кнопку `Accept`. Файлы в проекте автоматически обновились.


## 7. Анализ и исправление ошибок выполнения
*   **Магия:** Cursor успешно сгенерировал полный скрипт с интерактивным интерфейсом. Программа запустилась и чётко запросила путь к файлу.
*   **Боль:** При запуске `python chatbook.py` я по ошибке **ввела команду запуска `python chatbook.py`** в ответ на запрос программы "Введите путь к текстовому файлу". Это привело к ошибке `FileNotFoundError: Файл не найден: python chatbook.py`. Я не сразу поняла, что программа ждёт не команду, а имя файла с данными.
*   **Решение:** Я осознала логику работы консольного интерфейса: сначала запускается скрипт (`python chatbook.py`), а потом **внутри работающей программы** нужно ввести имя файла для анализа (например, `narnia.txt`). Я перезапустила скрипт и, когда программа снова задала вопрос, просто нажала `Enter`, чтобы выбрать файл по умолчанию (`narnia.txt`). После этого скрипт заработал корректно.

## 8. Запуск и типичная ошибка новичка
*   **Магия:** Код запустился, интерфейс работает, программа корректно запрашивает файл.
*   **Боль:** В ответ на запрос "Введите путь к файлу" я по ошибке ввела команду запуска скрипта `python chatbook.py`, что привело к ошибке `FileNotFoundError`.
*   **Решение:** Поняла, что программа ожидает имя файла с данными (например, `narnia.txt`), а не команду. Повторно запустила скрипт и при запросе просто нажала Enter для выбора файла по умолчанию.

## 10. Финальное тестирование и выводы: MVP готов!
*   **Магия:** **Прототип ChatBook полностью работоспособен!
** Все ключевые цели достигнуты:
    1.  **Чтение данных:
** Скрипт корректно читает `.txt` файл (в моём случае `narnia.txt`).
    2.  **Обработка текста:** Длинный текст автоматически разбивается на логические части.
    3.  **Поиск и ответ:** Функция `find_answer` находит в тексте релевантные фрагменты по ключевым словам из вопроса.
    4.  **Интерфейс:
** Удобный консольный интерфейс с циклом «вопрос-ответ».
    *   **Конкретный пример:
** На вопрос 
*«Что сказала Сьюзен про шкаф?»
* по тексту 3-й главы система мгновенно вернула отрывки, где упоминается этот персонаж.
*   **Боль (Главное ограничение MVP):** Алгоритм реализует **простой лексический поиск**.
 Он ищет буквальные совпадения слов, поэтому ответ — это 
**готовая цитата** из текста, а не сформулированный вывод. 
Эффективность сильно зависит от формулировки вопроса.
*   **Решение и итог:
** Задание по созданию 
**работающего прототипа (MVP)
** выполнено. 
Код является отличной основой.
 Для улучшения продукта следующими шагами будут: 
1) замена простого поиска на **семантический (векторный) поиск**; 
2) подключение **языковой модели (LLM)** для генерации связных ответов.
